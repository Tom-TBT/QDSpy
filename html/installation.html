Getting ready¶

Installation¶

  1. Install a current python distribution (64-bit, Python 3.4.x), e.g. Anaconda. The following steps were tested with Anaconda3 - using a different distribution may require the installation of additional packages.

    Note

    Python 3.5.x is currently not recommended, because a fully functional QDSpy requires the hidapi package and that is not yet not available for the latest Python version (for details, see below).

    You will need to start a windows shell can be started using cmd.exe or powershell.exe. The latter is the better alternative, as it supports, among other things, a Unix-like syntax.

  2. Anaconda currently does not come with PyOpenGL, therefore execute in the shell:

    pip install pyopengl

    This does not nescessarily install the OpenGL extension GLUT. In most cases QDSpy does not need GLUT. If, however, an error occurs because GLUT is missing, install GLUT following the instructions found here.

  3. Install the pyglet extension by executing in the shell:

    pip install pyglet

  4. If you plan to use the LightCrafter 4500 not only as a displaying device but also want to configure it via QDSpy (via USB), then you will need the hidapi package.

    Python 3.4.x:

    Just try to run QDSpy - the pre-compiled library hid.pyd provided in ...\QDSpy\Devices should work (tested with Python 3.4.x, Windows 8.1 64-bit).

    If this does not work, the hidapi package needs to be installed. Most likely, it cannot be installed via pip (see below) - this is because installing hidapi under Python 3.4.x via pip requires the Microsoft Visual C++ 10.0 compiler being present, and unfortunately there is no easy way of getting the needed components installed (so far only for Python 2, Microsoft provides a convinient installer).

    1. Install Microsoft Visual C++ 2010 (Express is sufficient)

    2. Install Microsoft SDK 7.1 (Windows 7)

    3. Create under C:\Program Files (x86)\Microsoft Visual Studio 10.0\VC\bin\amd64\ a .bat file called vcvars64.bat with the following content:

      "C:\Program Files\Microsoft SDKs\Windows\v7.1\Bin\SetEnv.cmd" /x64

    4. Now pip should work:

      pip install hidapi

    Python 3.5.x:

    For Python 3.5.x, installation of hidapi now requires Microsoft Visual C++ 14.0. Instructions to be added soon.

  5. If you plan to use a digital I/O card from Measurement Computing, you need to install their Universal Library (the 64-bit version). Note that you’ll need to reboot before the library is available to QDSpy. Enable the support for digital I/O in the Configuration file:

    [Timing]
...
bool_use_digitalio = True
int_digitalio_board_num = 0
int_digitalio_device_num = 1

  6. If you need gamma correction (currently Windows-only): Depending how strongly the color presentation needs to be changed, you may have to define/change a registry key that tells Windows normally to prevents too extreme color changes by the user. For details on gamma correction, see here.

    To deactivate Windows’ safety measures, write the following text in a file with the extension .reg and double-click it:

    [HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ICM]
"GdiIcmGammaRange"=dword:00000100

    To reactivate Windows’ safety measures:

    [HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ICM]
"GdiIcmGammaRange"=dword:00000000

    You may have to reboot for this change to become active.

    Note that QDSpy currently uses Windows’ SetDeviceGammaRamp function to change gamma. The advantage is that it is very simple to use, however, a side effect is that all connected screens are affected.

    Warning

    With the registry key set as described above, you can accidentally turn all screens unreadable!!

  7. Copy the content of the .zip file into a folder (e.g. C:\User\QDSpy), keeping the internal folder substructure intact.

  8. It is important to add the path to the QDSpy main folder to the PYTHONPATH environment variable. If this variable does not yet exist, it needs to be created. There are several ways to do this but the easiest way is probably using Spyder, which comes with the Anaconda package.

    1. run Spyder (note that the first start can take some time).
    2. in the menu, select Tools, then PYTHONPATH manager
    3. add the path to the QDSpy folder and press the synchronize button.

    Note that open powershells need to be restarted to acknowledge this change.

Running the GUI version of QDSpy¶

Open a windows shell (powershell.exe) and change to the QDSpy folder. Run the GUI version of QDSpy:

Windows PowerShell
Copyright (C) 2014 Microsoft Corporation. Alle Rechte vorbehalten.

PS C:\Users\Thomas\QDSpy3> python QDSpy_GUI_main.py

Alternatively, use the included batch file QDSpy.bat to start the GUI version of QDSpy. For instance, create a link to this batch file on the desktop; change the icon of this linke to QDSpy.ico`.

Using the batch file is equivalent to

PS C:\Users\Thomas\QDSpy3> python QDSpy_GUI_main.py --gui

The optional command line parameter --gui directs all messages (except for messages generated when scripts are compiled) exclusively to the history window of the GUI. The parameter is ignored when starting stimulus files directly from the shell.

Using QDSpy with the shell¶

Another way to compile and run stimulus scripts is by using the windows shell.

  1. Start a shell and change to the folder that contains the stimulus scripts:

Copyright (C) 2014 Microsoft Corporation. Alle Rechte vorbehalten.

PS C:\Users\Thomas> e:
PS E:\> cd .\user\Dropbox\QDSpy\QDS_stimuli
PS E:\user\Dropbox\QDSpy\QDS_stimuli>

  2. Compile a script (for details on writing QDS stimulus scripts, see Writing QDSpy stimulus scripts):
